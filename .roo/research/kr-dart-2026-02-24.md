## B2. Verbatim Endpoint Specifications

**Source**: DART Api doc, lines 1-44

### Authentication:
```
# COPIED VERBATIM from DART Api doc
| Parameter    | Name                  | Type       | Required |
|--------------|-----------------------|------------|----------|
| crtfc_key   | API authentication key | STRING(40) | Yes      |
```

### Company Info endpoint:
```
GET https://opendart.fss.or.kr/api/company.json?crtfc_key=KEY&corp_code=CODE
```

### Financial Statements endpoint:
```
GET https://opendart.fss.or.kr/api/fnlttSinglAcntAll.json
Parameters:
  - crtfc_key: API key
  - corp_code: DART corporate code
  - bsns_year: Business year (YYYY)
  - reprt_code: Report code (11011=annual, 11014=Q3, 11012=semi, 11013=Q1)
  - fs_div: Financial statement division (CFS=consolidated, OFS=separate)
```

### Disclosure List endpoint:
```
GET https://opendart.fss.or.kr/api/list.json?crtfc_key=KEY&page_count=100
```

### Executive Status endpoint:
```
GET https://opendart.fss.or.kr/api/exctvSttus.json
Parameters:
  - crtfc_key: API key
  - corp_code: DART corporate code
  - bsns_year: Business year
  - reprt_code: Report code
```

### Error Codes:
```
# COPIED VERBATIM from DART Api doc
- 000: Normal (success)
- 010: Not a registered key
- 011: Key suspended/temporarily unusable
- 013: No data/viewable
- 020: Call limit exceeded (often ~20,000/day)
- 100: Invalid field value
```

## B3. Verbatim Response Fields

**Source**: DART API response format (from our direct API code)
```
fnlttSinglAcntAll.json response fields:
- account_nm: Account name (Korean, e.g., "매출액" = Revenue)
- thstrm_amount: Current period amount
- frmtrm_amount: Prior period amount
- rcept_no: Receipt/filing number
- bsns_year: Business year
- reprt_code: Report code
```
